# Sembly MD šablona

> Vestavěná do n8n workflow `sembly-to-cowork.json` (Code node "Format → Markdown"). Data přicházejí přes **Webhook** ze Sembly Custom Automation (ne přes Sembly REST API). Po prvním doručení případně uprav mapování polí v Code uzlu podle skutečného JSON.

## Šablona

```markdown
# <title> — <YYYY-MM-DD HH:MM>

**Účastníci**: <jméno1>, <jméno2>, ...
**Trvání**: <X> min
**Sembly**: <permalink>

## Shrnutí

<auto-generated summary z Sembly>

## Akční body

- [ ] <action item 1>
- [ ] <action item 2>
...

## Plný transkript

<full transcript text>
```

## Naming souboru

`YYYY-MM-DD-HHMM-<title-slug>.md`

- `YYYY-MM-DD-HHMM` z `meeting.start_time`
- `title-slug` = NFC, zachová diakritiku; mezery → `-`; odstraní jen znaky nebezpečné v názvu souboru (`\ / : * ? " < > |` atd.); max 50 znaků

## Mapping Sembly fields

| Sembly API field | Použito v MD |
|------------------|--------------|
| `meeting.title` | H1 + filename slug |
| `meeting.start_time` | datum + čas v hlavičce + filename prefix |
| `meeting.attendees[].name` | Účastníci |
| `meeting.duration_minutes` | Trvání |
| `meeting.url` / `meeting.permalink` | Sembly |
| `meeting_transcription` (Sembly automation webhook) / `transcript` | Plný transkript |
| `summary` (text) | Shrnutí |
| `action_items[].text` | Akční body |

Pokud Sembly API tvoje verze používá jiné field names, oprav v Code nodu (mapping je v kódu jasný).

## Úpravy v n8n

1. Otevři workflow v n8n
2. Code node "Format → Markdown"
3. Uprav textové sekce podle potřeby
4. Save & Activate

Třídění do témat (`AGENDA/*.md`) řeší skill `agenda-triage` / `agenda-capture` — v .md už není automatický „suggested topic“.
